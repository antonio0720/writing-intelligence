# Genre Pack: Email

## Weighting Adjustments

| Category | Weight Modifier | Reason |
|---|---|---|
| Density | +5 | Every sentence costs the reader's time and goodwill |
| Clarity | +5 | Misread emails cause real damage |
| Audience Fit | +3 | Tone miscalibration in email is immediately felt |
| Rhythm | -3 | Functionality over musicality |
| Memorability | -5 | Emails are tools, not artifacts |

## Architecture

### Subject Line
- States the action or decision needed
- Under 8 words
- Never vague ("Quick question" / "Following up" / "Thoughts?")
- Good: "Decision needed: Q3 budget by Friday" / "Draft attached for your review"

### Opening Line
- State the purpose immediately. No "Hope you're well" filler.
- If relationship requires warmth, one sentence max, then purpose.
- "Sending the revised proposal — key changes are in Section 3."

### Body
- One email = one purpose. If you have two purposes, send two emails.
- If the email requires action: state the action, the owner, and the deadline.
- If the email provides information: state the key takeaway first, detail second.
- Bullet points only for genuinely parallel items. Not for padding.
- Bold the action item or decision point so it's scannable.

### Closing
- State the next step: who does what by when.
- "Let me know if you have questions" only if you genuinely expect questions.
- No "Cheers" / "Best" / "Warmly" unless it matches your real communication style.

## Common Failures

- Burying the ask in paragraph 3
- Emails that require the reader to infer what's being asked
- Overly formal language between people who know each other
- Overly casual language in professional contexts
- "Per my last email" (passive aggression)
- Reply-all when reply suffices
- Sending information emails that should be documents
